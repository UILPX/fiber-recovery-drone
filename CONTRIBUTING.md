# 开发协作规范

## 工作方式

- 先记录场景和风险，再设计机构和算法。
- 任何会增加牵引力、飞行接触或现场自主性的改动，都需要同步更新安全文档。
- 原型测试优先在封闭场地进行，所有现场测试都需要单独评审。
- 软件、硬件、操作流程和测试记录保持同步，避免“能跑但不可用”的原型。

## 文档约定

- `docs/` 存放概念、需求、架构、路线图和安全边界。
- `hardware/` 存放机械、电气、传感器和接口设计。
- `software/` 存放控制、感知、任务编排和操作台设计。
- `operations/` 存放现场流程、检查清单和培训材料。
- `tests/` 存放验证计划、测试记录和复盘。

## 变更检查

提交前确认：

- 是否改变了安全边界。
- 是否影响无人机载荷、飞行风险或末端工具。
- 是否影响地面机器人最大牵引力或急停能力。
- 是否需要新增测试场景。
- 是否需要更新操作流程。

## 贡献限制

本项目不是开放许可项目。提交 issue、pull request 或设计建议并不自动授予任何人制造、商业化、部署、再发布或衍生使用的权利。

不会接受以下贡献：

- 增加武器化、破坏、目标指示、监控或强制执法能力。
- 绕过安全停机、人工监督、现场合规或飞行合规。
- 提供未经排查危险区域的实际操作指南。
- 增加高力牵引、切割、破拆、爆破或危险物处理能力。
- 删除或弱化许可证、安全边界和负责任使用说明。
